## 一、整体架构

本方案基于 **CoAT（Chain-of-Action-Thought）** 行动思维链范式，融合 **MobileAgent** 的任务规划、**SeeClick** 的反思校正、**AppAgent** 的自适应滚动等前沿技术，构建了一个具备结构化推理、自我校正和长程任务管理能力的 GUI Agent。

```
┌─────────────────────────────────────────────────────────┐
│                    Agent.act() 主流程                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌──────────┐    ┌──────────────┐    ┌───────────────┐  │
│  │ 首步初始化 │───→│ 任务子目标分解 │───→│ APP名称推断    │  │
│  └──────────┘    └──────────────┘    └───────────────┘  │
│                           │                              │
│  ┌──────────┐    ┌───────▼────────┐    ┌──────────────┐  │
│  │ 截图哈希  │───→│ 防循环检测      │───→│ 构建Messages  │  │
│  │ + stuck  │    │ (动作+截图双重) │    │ (CoAT多轮)    │  │
│  └──────────┘    └────────────────┘    └──────┬───────┘  │
│                                                │          │
│  ┌──────────────────────┐    ┌─────────────────┘          │
│  │ 调用 VLM API (重试)   │◄───┘                          │
│  └──────────┬───────────┘                                │
│             │                                             │
│  ┌──────────▼───────────┐    ┌────────────────────────┐  │
│  │ 多格式输出解析        │───→│ 动作验证与修正          │  │
│  │ (标准/函数/段落/关键字) │    │ (坐标校验+摇摆检测+    │  │
│  └──────────────────────┘    │  反思注入+计划推进)      │  │
│                               └───────────┬────────────┘  │
│                                           │               │
│                               ┌───────────▼────────────┐  │
│                               │ AgentOutput            │  │
│                               │ (action + parameters)   │  │
│                               └────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 技术选型依据

| 技术方案 | 来源论文 | 采用原因 |
|---------|---------|---------|
| CoAT 思维链 | CoAT (复旦 DISC) | 仅用 Prompt 优化即可显著提升 STOP 准确率 +19% |

## 二、核心算法模块

### 2.1 CoAT 结构化推理

**原理**：强迫模型在输出动作前进行"观察→思考→行动"的三段式推理。

每一步，Agent 构建 CoAT 格式的多轮对话 messages 传入 VLM：

```
[system]  角色定义 + 操作规范 + 屏幕分区坐标参考
[user]    任务指令 + APP名称提示（首步）
[assistant] 好的，我来一步步完成这个任务。
[user]    操作历史摘要（含每步是否生效）
[assistant] 我继续执行下一步操作。
[user]    当前截图 + 子目标进度 + 反思提示 + 循环警告
```

**System Prompt 空间感知增强**：内嵌屏幕九宫格坐标参考，帮助模型精确定位 UI 元素：

```
屏幕分区参考：
- 状态栏 y:0~30 | 顶部导航/搜索框 y:30~120
- Tab栏/分类 y:120~200 | 内容区 y:200~850
- 底部导航栏 y:900~1000
- 水平：左 0~333 | 中 333~667 | 右 667~1000
```

**设计考量**：
- System Prompt 控制在约 500 字，平衡信息量与 token 消耗
- 图片编码使用 JPEG 格式（比 PNG 小 3~5 倍），大幅降低 input token

### 2.2 任务子目标分解（Plan-Execute）

**原理**：借鉴 MobileAgent 的 Plan-then-Execute 范式，首次调用时用轻量级 LLM 请求将任务分解为 3~8 步子目标。

**流程**：
1. 首次 `act()` 调用时，发送简洁的规划 prompt
2. 模型返回步骤列表（解析编号前缀，过滤无效行）
3. 后续每步携带当前子目标和后续方向

**Prompt 示例**：
```
[任务进度] 当前子目标 (3/7): 「输入火锅搜索」

后续目标: 点击搜索按钮 → 筛选评分4.5以上 → ...
```

**计划推进策略**：
- OPEN 动作（打开APP）→ 自动推进一步
- TYPE 动作（输入文字）→ 自动推进一步
- CLICK/SCROLL → 不自动推进（需要观察效果后再决定）

**退化机制**：如果计划生成失败（API 异常、解析失败），返回空列表，Agent 自动退化为无计划模式。

### 2.3 防循环检测与反思校正

#### 2.3.1 双重循环检测

统一使用 `stuck_count`（截图连续不变次数）管理所有截图变化检测：

| 检测条件 | 阈值 | 触发行为 |
|---------|------|---------|
| 连续相同动作+参数 | 3 步 | 警告 + 强制重置计数 |
| 截图连续不变 | 2 次 | 警告 + 注入反思提示 |
| CLICK↔TYPE 摇摆 | 4 次 | 强制改为 SCROLL |
| 连续 CLICK 无效 | 4 次 | 强制改为 SCROLL |

#### 2.3.2 具体失败反思（SeeClick）

当 `stuck_count >= 2` 时，不使用泛泛的"请换一种策略"，而是**分析具体失败原因并给出针对性建议**：

```
[操作失败分析] 你连续 3 次操作后页面没有变化。上次点击位置是 [500, 300]，
该位置可能不是可交互元素，或者目标元素在当前视口之外。
建议策略：1) 仔细阅读截图中的文字内容 2) 尝试向下滚动 3) 确认搜索框已激活再输入
```

针对 CLICK/SCROLL/TYPE 三种动作分别给出不同的失败原因分析和策略建议。

### 2.4 屏幕变化感知

**原理**：借鉴 MobileAgent 的屏幕 diff 思想，在历史记录中标记每步操作是否导致了页面变化。

**实现**：
1. 每步计算截图 MD5 哈希
2. 与上一步哈希对比，得到 `page_changed` 标记
3. 在历史摘要中附带效果描述

```
步骤2: [思考] 需要搜索火锅
       [操作] 点击位置 (100, 200)（页面未变化）

步骤3: [思考] 搜索框已激活
       [操作] 输入文字 "火锅"
```

模型能看到每步操作是否实际生效，从而做出更明智的后续决策。

### 2.5 自适应滚动量

**原理**：借鉴 AppAgent 的探索策略，根据滚动效果动态调整滚动幅度。

| 状态 | 调整 | 原因 |
|------|------|------|
| 滚动后页面未变 | 幅度 -100（最小 200） | 可能到达页面边界 |
| 滚动后页面变化 | 幅度 +50（最大 400） | 滚动有效，逐渐恢复 |

所有 SCROLL 回退（摇摆检测、无效点击、解析失败）统一通过 `_make_scroll_params()` 生成参数，支持 `direction="down"/"up"`。

### 2.6 历史轨迹管理

**三级记忆压缩策略**：

| 范围 | 展示方式 | 示例 |
|------|---------|------|
| 最近 5 步 | 完整展示（含思考+动作+效果） | 步骤3: [思考]... [操作] 点击... |
| 更早的步骤 | 压缩为动作统计摘要 | 步骤1-2（已完成）：CLICK 1次、OPEN 1次 |
| 子目标进度 | 简要提示 | 当前子目标 (3/7) + 后续方向 |

每步思考内容截断至 200 字，避免 token 膨胀。

### 2.7 动作验证与修正

后置校验 pipeline，确保输出格式合法：

```
原始动作 → [OPEN名称映射] → [CLICK坐标校验+边界内缩] → [摇摆检测]
         → [连续CLICK检测] → [最终动作]
```

**关键校验项**：

| 校验 | 规则 |
|------|------|
| OPEN 名称 | 别名表精确匹配 → 包含匹配 → 指令关键词推断 |
| CLICK 坐标 | [0, 1000] 裁剪 + 边界 <5 或 >995 时内缩 10 |
| TYPE 区分 | 若 TYPE 文字与上次不同，不计入 CLICK↔TYPE 摇摆 |

### 2.8 多格式输出解析

支持 4 种模型输出格式的逐级回退解析：

```
标准格式  CLICK:[500, 300]
    ↓ 失败
函数格式  click(500, 300)
    ↓ 失败
段落格式  从"### 操作执行"部分提取
    ↓ 失败
关键字格式  从自然语言描述中匹配 "打开..."/"输入..."
```

每种格式内使用正则表达式匹配，解析失败时自动重试（最多 2 次），全部失败则回退为 SCROLL。

### 2.9 APP 名称映射

三级映射策略，覆盖常见 APP 的别名和变体：

```
精确匹配 → 别名表反向查找 → 包含匹配 → 指令关键词推断 → 返回原名
```

示例：`"美团外卖" → 精确匹配 → "美团"`；`"tiktok" → 精确匹配 → "抖音"`

## 三、创新点

### 3.1 融合多论文优势的单 Agent 架构

将 MobileAgent 的多 Agent 协作理念（规划/决策/反思）**压缩到单 Agent 的 Prompt 管理中**，避免了多倍 API 调用的开销，同时保留了结构化推理的优势。

### 3.2 具体化反思而非泛化警告

区别于传统方法中泛泛的"请尝试不同策略"，本方案根据**最近操作类型和失败模式**生成具体的失败分析。例如：CLICK 失败时提示"目标元素可能在视口外"，SCROLL 失败时提示"可能到达页面边界，尝试反向滚动"。

### 3.3 操作效果感知闭环

在历史记录中附带 `page_changed` 标记，使模型能够理解"我做了什么 + 是否有效"，形成类似 MobileAgent 自愈机制的轻量级闭环，但不增加额外 API 调用。

### 3.4 自适应滚动策略

借鉴 AppAgent 的自适应探索思想，滚动无效时自动减小幅度（避免在页面边界浪费步数），有效时逐渐恢复，提高搜索效率。

## 四、项目结构

```
├── doc/
│   └── 算法设计说明文档.md    # 本文档
│
└── src/
    ├── agent_base.py          # Agent 基类（主办方提供，禁止修改）
    ├── agent.py               # Agent 派生类（核心实现，约 1070 行）
    ├── utils/                 # 辅助工具模块
    │   ├── __init__.py
    │   └── image_utils.py     # 图像编码/解码工具
    └── requirements.txt       # Python 依赖清单
```

### agent.py 模块结构

```
Agent (继承 BaseAgent)
├── act()                          # 核心入口：每步调用的主方法
├── _validate_and_correct_action() # 动作后置验证 pipeline
├── _make_scroll_params()          # 自适应滚动参数生成
├── _clamp_click_coordinates()     # 坐标范围校验 + 安全边界内缩
│
├── APP名称管理
│   ├── _resolve_app_name()        # 三级 APP 名称映射
│   └── _infer_app_from_instruction()  # 从指令推断目标 APP
│
├── 消息构建
│   ├── _build_messages()          # CoAT 多轮对话构建
│   ├── _build_system_prompt()     # 空间感知增强 System Prompt
│   ├── _build_current_prompt()    # 当前步提示（含进度+反思+警告）
│   └── _build_history_summary()   # 历史轨迹压缩摘要
│
├── 输出解析
│   ├── _parse_action()            # 多格式逐级回退解析
│   ├── _parse_standard_format()   # CLICK:[x,y] 等标准格式
│   ├── _parse_function_format()   # click(x,y) 函数格式
│   ├── _parse_section_format()    # 从"### 操作执行"部分提取
│   ├── _parse_keyword_format()    # 自然语言关键字兜底
│   └── _extract_thought()         # 思考内容提取
│
├── 防循环
│   ├── _detect_loop()             # 双重循环检测
│   ├── _compute_image_hash()      # 截图 MD5 哈希
│   └── _format_step_description() # 操作描述（含效果标记）
│
├── 子目标分解
│   ├── _generate_task_plan()      # LLM 任务分解
│   └── _advance_plan_step()       # 计划自动推进
│
└── 反思机制
    └── _build_reflection_prompt() # 具体失败分析生成
```
