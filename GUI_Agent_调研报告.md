## 二、各项目核心可借鉴点

### 2.1 Mano-P：Think-Act-Verify 三阶段闭环

**核心理念**：每一步操作不是简单的"看图→输出动作"，而是三阶段推理：

```
Think（思考）→ 推理当前状态与目标的差距
Act（行动）→ 执行具体操作
Verify（验证）→ 操作后验证结果是否符合预期
```

**可借鉴到比赛的要点**：

1. **在 Prompt 中强制要求"思考"环节**：让模型先描述"当前界面看到了什么"、"目标是什么"、"差距在哪里"，再输出动作
2. **验证意识**：虽然比赛环境无法真正执行验证，但可以在 Prompt 中培养模型的"验证意识"——输出动作时同时描述"预期结果"
3. **长任务自愈**：模型在长任务中不会因单步偏差导致后续全部跑偏

---

### 2.2 MobileAgent：多Agent + 自愈机制

**架构设计**（四Agent协作）：

| Agent | 职责 | 比赛中可简化为 |
|-------|------|---------------|
| 感知模块 | 截屏 → UI描述 + 坐标 | 用一次VLM调用 + 结构化Prompt |
| 规划代理 | 用户指令 → 子步骤序列 | 首次调用时用LLM做任务分解 |
| 决策代理 | 截屏 + 历史 → 动作预测 | 核心Agent逻辑 |
| 反思代理 | 对比执行前后 → 修正 | 截图变化检测 + 防循环 |

**可借鉴要点**：

1. **自愈机制**（最关键）：
   - 对比前后两帧截图，判断操作是否生效
   - 如果截图几乎没变 → 说明操作无效 → 触发修正策略
   - 默认重试3次，滑动操作重试5次

2. **记忆管理**：
   - 最近3步保留完整信息
   - 更早的步骤压缩为自然语言摘要
   - 维护关键实体（如已输入的搜索词、已选择的日期等）

3. **置信度机制**：置信度低于阈值（默认0.7）时触发自愈

### 2.3 CoAT：Chain-of-Action-Thought（行动思维链）

**核心发现**：仅用 1B 模型 + CoAT 方法 ≈ 18B 大模型的性能！

**四组件结构**（每步操作必须包含）：

| 组件 | 内容 | 示例 |
|------|------|------|
| 屏幕描述 | 当前屏幕的主要元素 | "当前是美团首页，顶部有搜索栏，中间有推荐商家列表" |
| 操作思考 | 分析当前状态与目标的关联 | "需要在搜索栏输入'北京南站附近酒店'" |
| 操作描述 | 正在操作的UI元素 | "点击顶部搜索框" |
| 操作结果 | 操作后的预期效果 | "进入搜索结果页" |

**消融实验关键结论**：
- 仅有"操作思考"输出 → 效果一般
- **屏幕描述 + 操作思考** → 性能立即提升
- **前一步结果 + 操作思考 + 操作描述** → 效果最佳
- **STOP（完成）动作预测准确率提升 19%**（60.12 → 79.17）

**可借鉴要点**：
1. 每步输出不仅包含动作，还必须包含"思考过程"
2. 前一步的操作结果作为下一步的输入
3. 这是对比赛最有直接价值的技巧——不改模型、只改Prompt就能大幅提升

### 2.4 SeeClick：坐标预测技巧

**核心发现**：GUI grounding 能力与下游 Agent 任务性能**直接正相关**

**坐标表示方式**：
- 坐标归一化为 **[0,1] 区间的浮点数**
- 比赛要求归一化到 [0,1000] 区间（一致思路）
- 直接将坐标作为自然语言文本输出，无需额外token

**关键发现**：
- 所有模型在 **Icon/Widget（图标/小组件）** 定位上最困难
- 文本类元素的定位准确率远高于图标类

**可借鉴要点**：
1. 让模型输出坐标时，先描述目标元素的**文本内容**，再输出坐标
2. 对于图标/按钮类元素，描述其**位置关系**（如"右上角"、"底部导航栏第二个"）

### 2.5 系统性GUI Agent方法总结

**三层记忆架构**：

| 层级 | 内容 | 比赛实现 |
|------|------|---------|
| 感知记忆 | 当前截图 | 直接传入VLM |
| 情景记忆 | 过去操作轨迹 | 维护 action_history 列表 |
| 程序记忆 | 常用操作模式 | 在Prompt中提供few-shot示例 |

**防循环检测**：
- 检测 `(Action, State)` 组合是否连续重复 3 次
- 截图哈希对比：连续两帧截图几乎相同时，说明操作无效

**打破策略**：
- 策略A：尝试滚动（找到隐藏元素）
- 策略B：点击空白处（关闭可能的弹窗/遮罩）
- 策略C：尝试返回（回退到上一级）

## 三、比赛落地方案

### 3.1 整体架构设计

```
MyAgent.act(instruction, screenshot, apps)
│
├── 1. 首次调用：任务分解（Task Decomposition）
│   └── 用LLM将复杂任务拆成子目标序列
│
├── 2. 截图预处理（可选）
│   └── 添加坐标参考信息到Prompt
│
├── 3. 构建结构化Prompt（借鉴CoAT）
│   ├── System Prompt：角色定义 + 操作规范
│   ├── 用户指令：原始任务 + 当前子目标
│   ├── 屏幕描述：留空由模型填充（CoT引导）
│   ├── 操作历史：最近5步的(思考→动作→结果)
│   ├── 防循环提示：如检测到循环，提示换策略
│   └── 当前截图（base64）
│
├── 4. 调用VLM → 解析输出
│   ├── 从回复中提取"思考过程"
│   ├── 从回复中提取 action + parameters
│   └── 解析失败时重试（最多2次）
│
├── 5. 防循环检测
│   ├── 记录每步的 action + screenshot_hash
│   └── 连续3步相似 → 触发突破策略
│
├── 6. 突破策略
│   ├── 尝试 SCROLL（找到隐藏内容）
│   ├── 尝试点击不同区域
│   └── 尝试 COMPLETE（适时结束）
│
└── 7. 返回 action + parameters
```

### 3.2 Prompt 设计核心模板（借鉴 CoAT + MobileAgent）

```
你是一个手机操作助手。你看到的是手机当前屏幕的截图。

## 任务目标
{user_instruction}

## 已完成的步骤
{history_steps}

## 当前子目标
{current_subtask}

## 请按以下格式思考并输出操作：

### 思考
1. 当前屏幕显示的是什么界面？主要有哪些元素？
2. 我需要完成什么操作？目标元素在哪里？
3. 应该执行什么类型的操作？

### 操作
[CLICK/TYPE/SCROLL/OPEN/COMPLETE]:[参数]

### 预期结果
执行此操作后，屏幕会发生什么变化？

## 操作规范
- CLICK: 点击元素，参数为 [[x, y]]，坐标范围 0-1000
- TYPE: 输入文字，参数为 ['要输入的内容']
- SCROLL: 滚动屏幕，参数为 [[起点x,y], [终点x,y]]
- OPEN: 打开应用，参数为 ['应用名']
- COMPLETE: 任务完成，参数为 []
- 坐标是相对坐标，图片宽高归一化至1000

## 注意事项
- 仔细观察截图中的文字内容和图标位置
- 描述目标元素周围的上下文，帮助定位
- 如果上一步操作没有生效（界面没变化），尝试其他操作方式
```

### 3.3 任务分解策略

首次调用时（history为空），先做任务分解：

```
请将以下任务分解为具体的子步骤：
"{instruction}"

每个步骤应该是一个可以在单次手机操作中完成的动作。
考虑常见的手机App操作流程。
```

然后按子步骤逐步推进，每完成一个子目标就切换到下一个。

### 3.4 防循环 + 突破策略

```python
def check_loop(history, current_screenshot_hash):
    """检测是否陷入循环"""
    recent = history[-3:]
    if len(recent) < 3:
        return False
    
    # 检查最近3步的动作是否相同
    actions = [h['action'] for h in recent]
    if len(set(actions)) == 1 and actions[0] != 'SCROLL':
        return True
    
    # 检查截图是否几乎没变化
    hashes = [h.get('screenshot_hash', '') for h in recent]
    if len(set(hashes)) == 1:
        return True
    
    return False

def breakout_strategy(context):
    """循环突破策略"""
    if 'search' in context.lower():
        return 'SCROLL', [[500, 700], [500, 300]]  # 向上滚
    else:
        return 'SCROLL', [[500, 500], [500, 800]]  # 向下滚
```

### 3.5 关键创新点（用于设计文档得分）

1. **CoAT 思维链融合**：将"行动思维链"引入比赛场景，让模型在每步操作前进行结构化思考
2. **多Agent理念简化版**：将规划、决策、反思三角色融入单一Agent的Prompt管理中
3. **智能防循环**：基于动作历史 + 截图相似度的双重循环检测
4. **动态任务分解**：复杂任务自动拆子目标，简单任务直接执行
5. **结构化历史记忆**：滑动窗口 + 语义摘要的记忆压缩策略

## 五、风险与注意事项

1. **模型固定**：比赛用火山引擎 Doubao，不能换模型，所有优化在Agent框架层
2. **API调用次数**：任务分解会额外消耗一次LLM调用，需要控制
3. **Prompt长度**：历史记录不能太长，否则会截断或影响模型效果
4. **输出解析**：模型输出格式可能不稳定，需要健壮的解析逻辑
5. **45步上限**：需要控制步数，避免在某个子任务上浪费过多步数
